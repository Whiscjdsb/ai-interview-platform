# 演示脚本

本文档适用于 5 到 8 分钟课程答辩或项目演示。

## 推荐演示顺序

1. 简短介绍项目定位和技术栈，约 30 秒。
2. 演示普通用户刷题闭环，约 3 分钟。
3. 演示 AI 面试助手，约 1.5 分钟。
4. 演示管理员后台，约 2 分钟。
5. 展示构建测试结果和后续规划，约 30 秒。

## 普通用户演示路线

### 1. 登录

打开前端地址，进入 `/login`。

演示话术：

```text
这里是用户登录页。登录成功后，前端会保存 JWT Token，并通过 Axios 自动携带 Authorization 请求头。
```

### 2. 查看题库

进入 `/questions`，演示关键词、题型、难度、标签筛选。

演示话术：

```text
题库列表支持多条件筛选和分页，普通用户在列表中不会看到正确答案和解析。
```

### 3. 查看题目详情

点击题目详情，进入 `/questions/:id`。

演示话术：

```text
题目详情只展示题干、题型、难度和标签，避免提前泄露答案。
```

### 4. 收藏题目

点击收藏按钮，再进入 `/favorites` 查看。

演示话术：

```text
收藏接口会避免重复收藏，收藏列表同样不会暴露正确答案和解析。
```

### 5. 提交答案

进入 `/practice/:id`，选择或输入答案并提交。

演示话术：

```text
客观题会自动判题，多选题会忽略选项顺序。只有提交成功后，页面才展示正确答案和解析。
```

### 6. 查看错题本

如果提交错误答案，进入 `/wrong-questions`。

演示话术：

```text
答错客观题后，系统会自动加入错题本。再次答对后，错题状态会变为已攻克。
```

### 7. 查看答题历史

进入 `/answer-history`，再打开某条详情。

演示话术：

```text
答题历史只展示当前用户自己的记录。详情页可以回顾用户答案、正确答案、解析和判题结果。
```

### 8. 使用 AI 面试助手

进入 `/ai-interview`，填写岗位、难度、题目数量，开始面试。

演示话术：

```text
AI 面试助手当前使用 Mock 规则生成题目和评分，项目预留了后续接入真实大模型的扩展位置。
```

### 9. 查看 AI 面试历史

提交面试后查看结果，再进入 `/ai-interview/history`。

演示话术：

```text
提交后的模拟面试会保存到 AI 历史记录，用户只能查看自己的记录。
```

## 管理员演示路线

### 1. 管理员登录

使用 `admin/admin123` 登录。

演示话术：

```text
管理员登录后，左侧菜单会额外显示管理后台。普通用户不会看到该菜单。
```

### 2. 查看后台首页统计

进入 `/admin`。

演示话术：

```text
后台首页展示用户、题目、收藏、错题、答题记录和 AI 面试记录总数。
```

### 3. 用户管理

进入 `/admin/users`，演示搜索和分页。

演示话术：

```text
用户管理列表不返回密码等敏感字段，只展示基础信息、角色和注册时间。
```

### 4. 题库管理

进入 `/admin/questions`，演示筛选。

### 5. 新增题目

进入 `/admin/questions/create`，填写题目、答案、解析和标签。

### 6. 编辑题目

从题库管理列表进入编辑页。

### 7. 删除题目

点击删除，展示确认弹窗。

演示话术：

```text
删除操作有确认弹窗，后端使用逻辑删除，避免直接破坏历史数据。
```

### 8. 查看收藏记录

进入 `/admin/favorites`。

### 9. 查看错题记录

进入 `/admin/wrong-questions`。

### 10. 查看答题记录详情

进入 `/admin/answers`，点击“查看详情”。

演示话术：

```text
管理员可以在详情弹窗中查看题目、用户答案、正确答案、解析和提交时间。
```

### 11. 查看 AI 面试记录详情

进入 `/admin/ai-interviews`，点击“查看详情”。

演示话术：

```text
管理员可以查看模拟面试总分、AI 总评和每道题的用户回答与点评。
```
